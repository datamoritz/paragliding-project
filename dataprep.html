1. Paragliding Flight Data

Paragliding flight data was obtained from the Paragliding Forum Leonardo XC server , which provides one of the largest public repositories of GPS flight tracks worldwide. Records include a downloadable .igc file with the full GPS trajectory, as well as separate metadata such as glider type, takeoff location, flight date, distance, and duration.

Access was automated with a Python script that logged into the platform using session cookies. The script iterated through Austrian flights for the years 2022-2024, downloading both the GPS track and metadata from the flight tables. Thus the data were stored in two formats: (1) a folder of raw .igc files (one per flight) and (2) a single raw_rows.json file containing the raw HTML content for each flight. The .igc files were downloaded by iterating through the cleaned metadata.csv file, i.e. duplicate and faulty flights were ignored.

The JSON file was then further processed and parsed using BeautifulSoup to extract structured metadata fields such as flight ID, pilot, takeoff site, glider model, flight date, duration, distances, and XC score. This structured data was assembled into a pandas DataFrame for cleaning and analysis. During preprocessing, dates were converted into a consistent date format, durations were transformed into minutes, distances were cleaned and converted into numeric values, and XC scores were stored as floating-point numbers. The pilot column was dropped to ensure data privacy (for this analysis names are irrelevant, also if a pilot made one or numerous flights). Erroneous text encoding was repaired. Duplicate flight entries, based on metadata, were removed. Flights shorter than 3 minutes, less than 3 km in total distance or less than 1 km in open distance were also removed (even though they may not be erroneous, their short contribution to the dataset as a whole is negligible). There were no missing values. The DataFrame was stored as a .csv file for later analysis. After loading the corresponding .igc files, the metadata was filtered to only keep flights with a matching GPS flight track (not all .igc files were made available for download). Of 4745 flights, 965 remain.

Dispayed is a snapshot of exemplar raw flight metadata and its structured representation after transformation. Each record corresponds to an individual paragliding flight and contains attributes such as the date, takeoff location, duration, open distance (straight-line), total distance (flown path), and the cross-country (XC) score, along with glider specifications. The XC score is a standardized performance metric calculated from the optimized flown distance (e.g., free distance, triangles) following OLC rules , allowing fair comparison of flights across conditions and pilots.

The distributions of flight duration, open distance, total distance, and XC score exhibit the expected right-skewness of paragliding data, with most flights being relatively short and only a few achieving long durations and distances. After log-transformation, the variables approximate unimodal distributions, improving comparability and reducing the impact of extreme values. The boxplots reveal outliers for open distance and XC score, however these are also to be expected (exceptional long-distance or competition flights).

GPS trajectory files were processed to first extract GPS data files (.igc) from the compressed .kmz archives. Longitude, latitude, and altitude values at 1 Hz resolution were extracted. To enrich the dataset with environmental context, terrain elevation was added from the Shuttle Radar Topography Mission (SRTM), allowing computation of altitude above ground level (AGL).

The data transformation is visualized below using a typical summer flight at Zell am See, Austria. The figure shows both a 2D trajectory map, with flight paths plotted over OpenStreetMap basemaps and annotated start/finish points, and a 3D trajectory map, which represents the full spatial trajectory including the altitude dimension.

Using the added terrain and altitude data, the figure below highlights the altitude above mean sea level (AMSL) as well as above ground (AGL) over time. The profiles demonstrate sustained thermalling phases with significant height gains, consistent terrain clearance, and a typical descent pattern toward landing.
